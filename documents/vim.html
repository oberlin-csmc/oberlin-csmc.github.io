Vim Night

Why vim?

You might be wondering what advantages vim provides over other editors like Komodo and IDEs like Eclipse. The main advantages of vim are not user-friendliness or feature density, but ubiquity and efficiency. Almost any Unix system will come with vim installed, so editing text with vim will have much less overhead than installing your favorite IDE. Furthermore, if you ever work remotely over a slow network connection, using a graphical editor can be impractical. Command-line text editors like vim have to send much less data over the network, which makes for a much more productive experience. And with practice, editing text with vim can become at least as efficient as using a graphical editor (think of the time you spend moving your hand to the mouse or the arrow keys.)

Starting vim

vim <filename>

For editing an existing file or creating a new one.

Modal editing

Insert mode

Vim starts in normal mode, so press i to enter insert mode. Look for the indicator in the status bar at the bottom. Now you can type text to insert it into the document. When you're done inserting text, press Escape to return to normal mode.

In insert mode, vim behaves a lot like a simple graphical editor. At first, it will be easiest to do all your editing in this mode. As you become more experienced with vim and normal mode, you'll begin to use insert mode only for inserting chunks of text.

Normal mode

This is the most powerful mode. In normal mode, you issue short (usually several keystrokes) commands to move through the file or cut and paste text. Commands are case sensitive.

Common commands include:

  • arrow keys to move a row or column at a time (or use h, j, k, l)
  • i to insert text at the cursor
  • o to open a new line below the current one, or (uppercase) O for above
  • d to cut (or delete) text
  • y to copy (yank) text
  • p to paste text
  • c to change text (delete and enter insert mode)

Most commands accept a movement specifier to indicate what portion of the text to apply them to. Most of these specifiers can be used without a command to simply move the cursor.

  • w: to the beginning of the next word
  • e: to the end of the current word
  • b: back to the beginning of the nearest word
  • 0: back to the beginning of the line
  • $: to the next newline (i.e. selects the rest of the line)
  • iw: from the beginning of the current word to the beginning of the next word
  • gg: to the beginning of the file
  • G: to the end of the file

Commands like delete, yank, and change can be applied to the entire line the cursor is on by doubling the command (i.e. dd, yy, and cc delete, copy, and change a full line, respectively).

To move backward through the text, I can press b repeatedly. If the cursor is at the beginning of a word, I can issue dw to delete it. If the cursor is elsewhere in the word, I can issue diw to delete it.

Another possible movement specifier is /, which is used to search through text. When you type /, you'll get a cursor in the status bar. Type a regular expression and press Enter. If you're using it without a command, this will move the cursor to the next instance of the expression in the text. Pressing n will move to the next occurrence, and N to the previous one. If you're using it with a command, the command will be applied to all the text between the cursor and the beginning of the matched text.

Other commands

From normal mode, enter a colon to begin some other important commands. In particular:

  • :w to write (save) your changes
  • :q to quit (:wq to save and quit, or :q! to quit without saving changes)
  • :%s/foo/bar/ to substitute each match of regular expression foo with bar. The % indicates the range in which to substitute—in this case the whole file. Leaving out the % will substitute on the current line. You can pass regex flags after the final /, most notably g to replace all instances on each line, as opposed to the default of one per line.

Visual mode

Visual mode allows the user to select some text to apply an action to. To enter visual mode, press v. The cursor will stay where it is, and moving it (e.g. with the arrow keys or a motion like $ or /foo) will adjust the range of the selection. When you're ready to apply an action, simply enter the action. For example, to delete a word in visual mode, you would position the cursor at the beginning of the word and press v, then e to move to the end of the word, and then d to delete. After the action, vim takes you back to normal mode. To return to normal mode without taking an action, press Escape.

Visual line mode is an alternative visual mode that selects entire lines. To enter visual line mode, type (uppercase) V. This is especially useful for cutting several lines at a time.

vimrc

On startup, vim reads options from the file .vimrc in your home directory (on Linux), if it exists. To customize vim's behavior, you can edit this file. Here are some lines you might want to include:

To enable automatic syntax highlighting:

syntax on

To tell vim to automatically indent (e.g. when you press Enter after typing an indented line):

set autoindent

To show tabs as four spaces:

set tabstop=4
set shiftwidth=4

What next?

It takes time to learn these commands and to become familiar with vim's modes. If you'd like to learn to use vim efficiently, you'll need to use it frequently. This means you should consider using it for writing code, and perhaps for some things you currently use a word processor for.

Find a "cheat sheet" for vim and bookmark it or save it. As you use vim, be aware of how many keystrokes you're using for a given task (note in particular how often you use the arrow keys—less is usually better). If you notice that you're spending a lot of keystrokes for a task and you suspect vim might have a command to simplify the operation, check the cheat sheet. If there's nothing there, do a web search. If you find a command that helps, redo the task using that command, and try to use the command in the future. This is how you'll become an efficient vim user.
